# Using ESLint as Formatter in Next.js

The work of Linter is to check the code quality and make sure that the code is written in a way that is easy to read and maintain but the work of Formatter is to fix the formatting of the code. We should use a proper Formatter like `Prettier` to format the code in our project but ESLint also provides support for formatting. We should not use a Linter to format the code because it slows down the development process.

- Next.js ESLint Configuration: https://nextjs.org/docs/app/api-reference/config/eslint
- ESlint Docs: https://eslint.org/docs/latest/
